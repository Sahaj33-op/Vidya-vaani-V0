## 1. Purpose

The purpose of **Vidya Vaani** is to provide **24/7 multilingual, conversational support** for students, parents, and administrators in colleges/universities.
It reduces repetitive queries, ensures equitable access to information in **native/regional languages**, and empowers staff by freeing them from routine workloads.

The chatbot will:

* Provide **instant answers** to FAQs such as fees, timetables, scholarships, and deadlines.
* Support **voice and text input** in multiple languages (English, Hindi, Gujarati, Marwari, etc.).
* Deliver a **seamless experience** on both college websites and popular messaging platforms (Telegram, future WhatsApp).
* Work in both **online** (cloud-enabled) and **offline** (local deployment with fallback translation) modes.
* Ensure **data privacy**, **maintainability**, and **scalability** across multiple institutes.

### üåê Multilingual & Language-Agnostic

* Automatic **language detection** (fastText) + translation (MarianMT/Google Translate API).

### üì¢ Proactive Notifications

## 3. How It Works (System Workflow)

1. **Data Ingestion**

   * PDFs, FAQs, and notices ‚Üí OCR (Tesseract) ‚Üí Parsing (spaCy) ‚Üí Embedding (SentenceTransformers) ‚Üí Stored in **Supabase/pgvector**.

2. **User Query Handling**

   * Input: Voice or text in any supported language.
   * Language detection (fastText) ‚Üí Translation if needed ‚Üí Intent detection (Rasa).
   * Semantic search (vector DB) retrieves relevant context.
   * Response generated via **LLM (Gemini/OpenAI)** + rules.
   * Answer delivered in **user‚Äôs original language** (via back-translation).

3. **Delivery Channels**

   * Web widget (React) embedded in college site.
   * Telegram bot for instant access.
   * Optional React Native offline app.

4. **Feedback & Escalation**

   * Student can rate answers.
   * If unresolved ‚Üí escalate to human staff.
   * Logs stored for admin review (via dashboard).
